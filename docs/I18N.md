# i18n

The plugin ships 6 locales:

| Code | Label |
|------|-------|
| `en` | English |
| `fr` | Français |
| `de` | Deutsch |
| `it` | Italiano |
| `es` | Español |
| `nl` | Nederlands |

## How it works

- All translations live in `src/i18n.ts` as flat `Record<string, string>` dicts — no runtime lib (no `i18next`, no `react-intl`).
- Default language is detected from `navigator.language.slice(0,2)`; falls back to `en`.
- User can override via the Language panel; choice persists in `config.json` as `language`.
- `makeT(lang)` returns a `t(key, vars?)` function with simple `{placeholder}` substitution.
- Missing keys fall back to English, then to the literal key string — UI never crashes on an untranslated string.

## Adding a key

1. Add the key to the English dict first (authoritative).
2. Add the same key to each of the other 5 locales.
3. Use `t("your.key", { var: 42 })` in components.

A missing translation in one locale silently falls back to English — safe, but run a quick diff before release:

```bash
node -e '
  const i18n = require("./src/i18n.ts"); // swap for a real ts runner
  // compare key sets
'
```

(Suggested improvement: split `src/i18n/{en,fr,de,it,es,nl}.ts` + `src/i18n/index.ts` for easier diffs and translator handoff — not done yet.)

## Adding a locale

1. Add the code to `Lang` union + `LANGS` + `LANG_LABELS` in `src/i18n.ts`.
2. Add the same code to `VALID_LANGS` in `config.py`.
3. Add the dictionary object.

## Translation style guide

- Keep keys short; describe role, not text: `btn.sync`, `toast.sync.title`.
- Keep placeholder names identical across locales: `{added}`, `{total}`.
- Prefer short strings — Decky panels are narrow (≈280 px). Long labels wrap ugly.
- Technical terms (`Proton`, `SteamGridDB`, `CEF`) remain in English in every locale.
